# CS 305 – Software Security: Artemis Financial

## Overview

This project focuses on performing a vulnerability assessment and applying secure software practices for a client, Artemis Financial. The purpose of this project was to identify security weaknesses in the software application and implement improvements to protect sensitive financial data and ensure system reliability. This work demonstrates how secure coding and vulnerability analysis are applied in real-world software development.

---

## Client Summary

Artemis Financial is a financial services company that requires secure software systems to protect customer and transaction data. The main issue the company wanted to address was identifying vulnerabilities in their existing application and improving its security to prevent cyberattacks, data breaches, and unauthorized access. Ensuring strong security is critical for maintaining trust and protecting sensitive financial information.

---

## Security Assessment and Performance

During the vulnerability assessment process, I identified several potential risks, including outdated dependencies and possible weaknesses in data handling. I used tools and manual analysis to evaluate these issues. Coding securely is important because it prevents common security threats such as injection attacks, data leaks, and unauthorized system access. Strong software security adds value to a company by protecting its assets, maintaining customer trust, and ensuring compliance with security standards.

---

## Challenges and Learning

One of the most challenging parts of this project was understanding how different vulnerabilities impact the system and how they can be exploited. However, this challenge helped me develop a deeper understanding of security risks and improved my ability to think critically when analyzing software systems. This experience strengthened my problem-solving skills and awareness of secure development practices.

---

## Improving Security Layers

To improve security, I updated dependencies, applied secure coding practices, and used automated tools such as OWASP Dependency-Check to identify vulnerabilities. I also focused on improving input validation and reducing potential attack surfaces. In the future, I would use additional tools such as penetration testing, static and dynamic analysis, and continuous security monitoring to assess vulnerabilities and select appropriate mitigation techniques.

---

## Ensuring Secure and Functional Code

After refactoring the code, I ensured that the application remained both functional and secure by testing it thoroughly and rerunning vulnerability scans. This helped confirm that existing issues were resolved and no new vulnerabilities were introduced. Continuous testing is essential to maintain both performance and security in software applications.
